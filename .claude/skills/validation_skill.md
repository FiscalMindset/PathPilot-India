# Validation Skill

## Purpose
Apply a rigorous cross-agent review rubric before the system claims completion.

## Use When
- the Reviewer Agent audits outputs
- a major phase concludes
- the final application architecture is assembled

## Validation Checklist
- original prompt has been correctly interpreted
- recommendations fit the user profile and India context
- frontend visibly exposes agent activity and trust indicators
- backend can support the designed observability experience
- roadmap is realistic and sequenced
- finance and access constraints are respected
- entrepreneurship path is optional and justified
- assumptions and uncertainties are visible

## Method
1. Review artifacts against the rubric, not subjective taste.
2. Flag contradictions and missing evidence first.
3. Separate blocking findings from enhancement ideas.
4. Route repairs to the narrowest responsible agent.
5. Require explicit closure notes for every blocking finding.

## Quality Bar
- No approval without observability-first UX.
- No approval when recommendations are untraceable.
- No approval when the system looks like a single-agent wrapper dressed up as multi-agent.
