# Frontend Generation Playbook

## Purpose
Guide the runtime generation of the PathPilot India frontend so it looks like a serious observability-first operations dashboard for opportunity guidance.

## Experience Priorities
- clarity of recommendation value
- clarity of agent process
- mobile-first adaptability
- trust and explainability
- visual distinction from a generic chat interface

## Required Screens Or Views
- onboarding and profile intake
- main run dashboard
- opportunity exploration and comparison
- roadmap and milestones
- mentor and community support
- scholarship and affordability guidance
- entrepreneurship branch when active
- wellness and pacing adjustments
- trace explorer and raw event drawer

## Responsive Layout Rules
- desktop: three-column operations layout plus bottom drawer
- tablet: collapsible side panels with persistent top metrics
- mobile: stacked cards with tabbed access to timeline, details, and recommendations

## Required UI Behaviors
- live status changes for agent states
- visible retries and warnings
- confidence badges
- source influence summaries
- drill-down from final synthesis into contributing artifacts

## Design Guardrails
- do not generate a chat-only shell
- do not hide trace details under developer-only controls
- do not make the dashboard visually sterile; it should feel intentional and modern
