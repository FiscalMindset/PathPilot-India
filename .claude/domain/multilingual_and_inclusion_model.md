# Multilingual And Inclusion Model

## Purpose
Ensure the generated application is designed for users across language comfort levels and social contexts within India.

## Required Product Behaviors
- multilingual onboarding path
- simple-language summaries of final recommendations
- support for mixed-language users in future localization
- explainability that does not depend on technical vocabulary
- UI hierarchy that remains legible on smaller devices

## Required Agent Behaviors
- Self-Discovery Agent must capture language comfort where inferable.
- Frontend Agent must design for localization readiness.
- Roadmap Planner Agent must avoid assuming English-only resources when affordable alternatives exist.

## Reviewer Checks
- is the generated app understandable to a first-generation or non-expert user?
- does the plan over-assume English fluency?
- are trust indicators and assumptions readable without technical jargon?
