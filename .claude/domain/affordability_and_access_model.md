# Affordability And Access Model

## Purpose
Prevent the system from generating elite-assumption roadmaps that ignore real-world access barriers common in the target user base.

## Constraint Dimensions
- tuition or certification cost
- device quality
- bandwidth reliability
- commute or geography barriers
- family obligations
- time available per week
- language comfort
- local ecosystem availability

## Required System Behavior
1. Every major recommendation must be evaluated for cost and access friction.
2. If a recommendation has high friction, the system must either:
   - justify it strongly, or
   - provide a lower-friction fallback
3. Roadmaps must explicitly account for time bandwidth.
4. Mentor and community recommendations must include accessible options.
5. Scholarship and grant discovery must connect directly to affordability pain points.

## Reviewer Checks
- Does the roadmap assume continuous high-speed internet?
- Does the path assume expensive programs without mitigation?
- Does the plan require metro-only access without alternatives?
- Does the system acknowledge language or confidence barriers?
