# Automation Depth Rubric

## Purpose
Ensure the system earns points for real automation rather than static architectural prose.

## Scoring Dimensions

### 5 - Excellent
- planning, research, synthesis, validation, repair, and final assembly are all automated stages
- runtime build manifests and generation scopes are explicit
- code generation occurs outside the blueprint repository
- the generated app can visibly demonstrate autonomous construction flow

### 4 - Strong
- most major stages are automated
- some deeper runtime behaviors are described rather than fully specified

### 3 - Adequate
- architecture is agentic, but final assembly feels under-specified

### 2 - Weak
- too much of the “automation” is implied instead of operationalized

### 1 - Failing
- blueprint contains static outputs or relies on manual intervention

## Mandatory Checks
- single-prompt initiation supported
- runtime generation protocol defined
- repair loops are not manual
